## How documentation is organized

1. **`docs/COMMANDS.md`** â€” Topic index, human-written explanations, DONE/PARTIAL status, and tables for major modules.
2. **`docs/reference/*.md`** â€” Deep dives per subsystem (window, render, physics, â€¦).
3. **`docs/API_CONSISTENCY.md`** â€” Generated list of every manifest overload and arity (`go run ./tools/apidoc`).
4. **`docs/reference/API_CONVENTIONS.md`** â€” Recommended verbs (`LOAD`, `SETPOS`, â€¦) across object types.
5. **Optional `description` fields** in `commands.json` â€” Extra inline help for tools; not required for every row.
